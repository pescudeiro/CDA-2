¿Qué es Product Matching?
Definición
Dado el mismo producto descrito por dos vendedores distintos, determinar si los registros corresponden al mismo ítem.
Es un problema de Record Linkage o Entity Resolution: identificar entidades del mundo real a partir de representaciones ruidosas y heterogéneas.

El challenge: títulos, marcas y precios se formatean de manera completamente distinta entre fuentes.
✓ Match — mismo producto
Table A: motu digital performer 5 competitive upgrade (mac)
Table B: motu digital performer dp5 software music production
✗ No Match — distintos productos
Table A: microsoft visio standard 2007 version upgrade
Table B: adobe cs3 design standard upgrade
⚠ Falso Positivo Potencial
[intuit] quicken 2007 premier
[intuit] quickbooks pro 2007 3 user

III
Sección 3
Transformaciones
y Feature Engineering
Cuatro frentes de preparación
1
Normalización de Texto
Lowercase · remoción de puntuación · colapso de whitespace. Columnas _norm preservan la versión cruda para trazabilidad.
2
Estandarización Numérica
Z-score sobre price con encoder unificado A+B. Winsorización del p99 para outliers extremos.
3
Codificación de Categóricas
Label encoding unificado A+B. En Walmart–Amazon: 1.972 marcas únicas, solo 400 en ambas tablas.
4
Generación de Features
word_overlap (Jaccard) · brand_match · category_match · price_diff · price_ratio · has_both_prices · len_diff
Validación: poder discriminativo de las features
Correlación de Pearson feature–label por dataset (train)
word_overlap
+0.43
Feature dominante. Consistente en los tres datasets (+0.37 a +0.44). Valida el enfoque textual.
brand_match
+0.12
Aporta especialmente en Amazon–Google. Menor efecto en los otros.
price_diff
–0.09
Signo correcto: productos matched tienen precios más similares.
Hallazgo: category_match no aporta en Walmart–Amazon (−0.01). Se mantiene por no dañar, se reporta.
Decisiones clave y próximos pasos
Entrega 3 — Modelado
1
Embeddings textuales
Sentence-transformers sobre el campo principal; agregar cosine_similarity como feature.
2
Baseline supervisado
Regresión logística + features actuales. Piso de F1 documentado.
3
Modelo expresivo
Gradient boosting o fine-tune de encoder. Comparación contra baseline.
4
Análisis de errores
Falsos positivos que word_overlap solo no detecta.
Dificultades identificadas
Desbalance 9:1
Plan: class_weight='balanced' o SMOTE en entrega 3.
Falsos positivos de marca
Office 2007 Standard vs Office 2007 Home: misma marca, mismo año, distinto producto.
Ruido textual entre fuentes
Matching exacto insuficiente. Justifica el uso de embeddings semánticos.
Decisiones de esta entrega
· Normalización textual prioritaria
· NaN en price sin imputar
· Label encoding unificado A+B
· Columnas crudas preservadas
